1. Single Responsibility Principle (SRP) adalah aturan dimana setiap kelas itu seharusnya hanya memiliki satu tanggung jawab saja. Dengan memisahkan tanggung jaawab ke dalam kelas kelas yang berbeda maka akan membuat code lebih muda untuk dipahami, diuji dan dikelola. 
2. Open/Closed Principle (OCP) dilanggar jika kode yang sudah ada harus diubah setiap kali ada fitur baru. Untuk menghindari hal tersebut maka kita dapat Menggunakan interface yang memungkinkan penambahan fitur baru tanpa mengubah kode lama.
3. Liskov Substitution Principle (LSP) dilanggar jika subclass tidak dapat digunakan sebagai pengganti parent class dengan aman. Untuk mengikuti prinsip ini, setiap subclass harus memenuhi kontrak yang didefinisikan oleh parent classnya.
